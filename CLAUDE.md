# 프로젝트 관리 대시보드 - 개발 가이드

## 프로젝트 개요
이 프로젝트는 프로젝트 관리 대시보드 애플리케이션의 프론트엔드입니다. React + TypeScript + Vite + Material-UI를 기반으로 구축되었습니다.

## 아키텍처 설계 및 컴포넌트 구조

### 생성된 컴포넌트와 설계 이유

#### 1. 메인 대시보드 페이지 (/)
**Why**: 프로젝트 관리의 중심 허브 역할을 하는 페이지가 필요합니다. 사용자가 한눈에 전체 프로젝트 현황을 파악하고 새로운 프로젝트를 생성할 수 있는 통합된 인터페이스를 제공합니다.

**What**: 
- 상단: 4개의 요약 카드 (전체/진행중/완료/지연 프로젝트 수)
- 중앙: 프로젝트 생성 폼
- 하단: 프로젝트 카드 리스트
- 레이아웃: Material-UI의 Grid 시스템을 활용한 반응형 디자인

#### 2. 프로젝트 생성 폼 컴포넌트
**Why**: 새로운 프로젝트를 체계적으로 생성하기 위한 전용 컴포넌트가 필요합니다. 사용자 입력 검증과 UX를 고려한 폼 설계로 데이터 품질을 보장합니다.

**What**:
- 프로젝트명 (TextField, 최대 100자)
- 설명 (TextArea, 최대 500자)
- 마감일 (DatePicker)
- 우선순위 (Select: 높음/보통/낮음)
- 담당자 선택 (Autocomplete)
- 실시간 폼 검증 및 토스트 알림

#### 3. 프로젝트 카드 리스트 컴포넌트
**Why**: 생성된 프로젝트들을 시각적으로 효과적으로 표시하고 관리할 수 있는 인터페이스가 필요합니다. 카드 형태의 레이아웃은 정보 스캔이 용이하고 직관적입니다.

**What**:
- 프로젝트 정보 카드 (Card 컴포넌트)
- 상태 배지 (Chip 컴포넌트)
- 진행률 바 (LinearProgress)
- 필터링 및 검색 기능
- 편집/삭제 액션 메뉴

#### 4. 대시보드 요약 카드 컴포넌트
**Why**: 프로젝트 현황을 한눈에 파악할 수 있는 KPI 대시보드가 필요합니다. 관리자나 팀 리더가 빠르게 상황을 인지하고 의사결정을 내릴 수 있도록 지원합니다.

**What**:
- 4개의 메트릭 카드 (전체/진행중/완료/지연)
- 아이콘과 색상을 통한 시각적 구분
- 전월 대비 증감률 표시
- 반응형 그리드 레이아웃

## 기술 스택
- **React 18**: 최신 React 기능 활용
- **TypeScript**: 타입 안정성 보장
- **Vite**: 빠른 개발 서버 및 빌드
- **Material-UI (MUI)**: 일관된 디자인 시스템
- **Emotion**: CSS-in-JS 스타일링

## 디렉토리 구조
```
src/
├── components/
│   ├── common/          # 공통 컴포넌트
│   └── ui/              # UI 컴포넌트
├── features/
│   ├── dashboard/       # 대시보드 관련 기능
│   └── projects/        # 프로젝트 관리 기능
├── hooks/               # 커스텀 훅
├── services/            # API 서비스
├── store/               # 상태 관리
├── styles/              # 글로벌 스타일
└── utils/               # 유틸리티 함수
```

## 개발 가이드라인

### 컴포넌트 개발 원칙
1. **단일 책임 원칙**: 각 컴포넌트는 하나의 명확한 역할을 가져야 합니다.
2. **재사용성**: 공통 컴포넌트는 props를 통해 다양한 상황에서 재사용 가능하도록 설계합니다.
3. **타입 안정성**: 모든 props와 상태에 대해 TypeScript 타입을 정의합니다.

### 상태 관리 전략
- **로컬 상태**: useState를 사용하여 컴포넌트 레벨에서 관리
- **전역 상태**: 프로젝트 데이터는 Context API 또는 Redux Toolkit 사용 고려
- **서버 상태**: React Query 또는 SWR을 활용한 캐싱 및 동기화

### 스타일링 가이드
- Material-UI 테마 시스템 활용
- 다크 모드 기본 적용
- 반응형 디자인 (모바일 우선)
- 일관된 색상 팔레트 및 타이포그래피

### 성능 최적화
- React.memo를 활용한 불필요한 리렌더링 방지
- useCallback, useMemo를 통한 함수 및 값 메모이제이션
- 코드 스플리팅을 통한 번들 크기 최적화
